Setting the ValueMember property isn't necessary. If you read about the SelectedValue property in the ComboBox docs is sure seems like if you set the ValueMember you should be able to use the ComboBox's SelectedValue property to determine the selected value (duh!). The docs are poorly written and SelectedValue only works when the Items property is databound to a list. That's more trouble than it's worth. Instead, as you'll see farther on down in the code, we'll avoid the the need for the SelectedValue property by using the SelectedItem property. 

        // comboboxFormType.ValueMember = 'IntegerValue'

This code is the functional equivalent of the two Items.Add lines above. It illustrates how effective a good constructor can be in helping keep code succinct. With a good constructor you can add a item with one line of code; without, you need four lines per item plus one for a ComboBoxItem declaration. Less code isn't always better--don't ever sacrifice clarity for brevity. In this case, though, the brevity isn't penalizing.

Declare and instance the NonModalForm. In AVR for .NET, 
you always need to instance a form before it is displayed. Depending 
on how frequently the form is displayed, and if you need its state to 
persist, it may be better to instance the form globally instead of

Display the form. Calling Show without any arguments displays the nonmodal form.
In this case, if the parent(ie, *This form) is minimized the child form (ie, NonModalForm)
is not minimized. 

    

    

      
        // NonModalForm.Show()

    

  

  

  

    

      

        #
      

      
The Show method has an overload that accepts one argument: the form to act as the child form's 
parent. The Microsoft Show docs don't show this overload--but 
the Visual Studio object browser does and its been there a long time. Curiously, the Show
method overload without an argument doesn't show in the object browser.




Calling Show with the argument *This, makes this form the child's form's owner. One 
of the things  this does is minimizes the child when the parent minimizes (which is 
probably what you want). 

    

    

      
        NonModalForm.Show(*This)

    

  

  

  

    

      

        #
      

      
AVR has a Show opcode you could also use to show a non-modal form. AVR's Show opcode 
behaves like the second Show method behavior discussed above--ensuring the child form 
minimizes with the parent. While there is no functional difference between 


NonModalForm.Show(*This) 


and 


Show NonModalForm


the Show opcode may be confusing to VB.NET and C# programmers. My advice is to stick 
with the Show method for that reason. That said, if you do, you do need to remember to
pass the *This argument if you want the child-follows-parent minimizing behavior discussed. 

    

    

      
        // Show NonModalForm
    EndSr
